# AI UI Intent Capture

## Summary

Article reference about designing AI interfaces that infer and refine user intent through contextual UI patterns instead of relying only on open-ended prompts.

## Key Points

- [How can AI UI capture intent?](https://uxdesign.cc/how-can-ai-ui-capture-intent-d7a6d0393ded) by Sharang Sharma explores contextual prompt patterns that surface likely intent while the user types.
- The core idea is that AI products should reduce prompt friction by helping users specify context through adaptive controls such as chips, dropdowns, sliders, segmented ranges, and inline suggestions.
- Search workflows can use contextual filters and defaults for predictable query types, such as terrain for running shoes, proximity and ambience for restaurants, source/time filters for financial queries, or audience/depth controls for explainers.
- File upload workflows can turn uploads into active entry points by suggesting actions based on file type, such as nested summarize/extract presets or lightweight ghost-text prompts.
- Content creation workflows can expose tone, style, format, camera angle, aspect ratio, and topical-tag controls to help users shape outputs without writing perfect prompts upfront.
- Useful principle: treat AI UX as intent refinement, not just chat input and response iteration.
